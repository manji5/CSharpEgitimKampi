# **Ders 35 Notları: `throw` ile Hata Fırlatma**

Selam Dostlar! Bu derste, sistemin hatalarını değil, kendi kurallarımıza uymayan durumları (Mantıksal Hatalar) nasıl yöneteceğimizi öğrendik.

### **1. `throw` Nedir?**

Programın akışını manuel olarak kesip, "Burada bir hata var!" sinyali göndermemizi sağlayan komuttur.

- **Neden Kullanırız?** Bir metodun veya özelliğin, geçersiz bir veriyle çalışmasını engellemek için.

### **2. Nasıl Kullanılır?**

Genellikle `new` anahtar kelimesiyle yeni bir Hata Nesnesi (Exception Object) oluşturup fırlatırız.

```csharp
// Genel Hata Fırlatma
throw new Exception("Bir hata oluştu.");

// Özel (Daha Doğru) Hata Fırlatma
throw new ArgumentException("Geçersiz bir değer girdiniz.");
```

### **3. Pratik Örnek: Bakiye Kontrolü**

Bir banka uygulamasında para çekerken bakiye yetersizse işlem yapılmamalıdır

```csharp
public void ParaCek(decimal miktar)
{
    // KURAL KONTROLÜ
    if (miktar > _bakiye)
    {
        // Hata fırlat! İşlemi durdur.
        throw new InvalidOperationException("Yetersiz Bakiye! İşlem yapılamaz.");
    }

    // Hata fırlatılırsa kod buraya ASLA gelmez.
    _bakiye -= miktar;
    Console.WriteLine("Para çekildi.");
}
```

**Kullanımı (try-catch ile):**

```csharp
try
{
    hesap.ParaCek(5000); // Bakiye yetersizse buradan fırlatılan hata...
}
catch (Exception ex)
{
    Console.WriteLine(ex.Message); // ...burada yakalanır: "Yetersiz Bakiye!..."
}
```

Bu yöntemle, hatalı işlemlerin gerçekleşmesini kesin olarak engellemiş oluruz.
