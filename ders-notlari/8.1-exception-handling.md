# **Ders 33 Notları: Exception Handling (Hata Yakalama) Giriş**

Selam Dostlar! Bu derste, programımızın aniden çökmesini (Runtime Error) engelleyen `try-catch` yapısını öğrendik.

### **1. Hata Türleri**

- **Syntax Error (Yazım Hatası):** Kod yazarken çıkan kırmızı çizgiler. Program hiç çalışmaz. (Örn: `;` unutmak).
- **Runtime Error (Çalışma Zamanı Hatası):** Program çalışırken oluşan hatalar. Programın çökmesine neden olur. (Örn: Kullanıcının sayı yerine harf girmesi, sıfıra bölme).

### **2. `try-catch` Yapısı**

Hata oluşabilecek kodları bir "güvenlik çemberine" almamızı sağlar.

Keşfet

- **`try` (Dene):** Hata çıkarma ihtimali olan kodları buraya yazarız.
- **`catch` (Yakala):** Eğer `try` bloğunda bir hata oluşursa, program çökmez, çalışmayı durdurup hemen buraya atlar.

```csharp
Console.Write("Sayı girin: ");

try
{
    // RİSKLİ BÖLGE
    int sayi = Convert.ToInt32(Console.ReadLine());
    Console.WriteLine($"Girdiğiniz sayı: {sayi}");
}
catch
{
    // GÜVENLİK AĞI
    // Hata olursa (örn: harf girerse) burası çalışır.
    Console.WriteLine("Hata! Lütfen geçerli bir sayı giriniz.");
}

Console.WriteLine("Program devam ediyor..."); // Program çökmediği için burayı görürüz.
```

### **3. Hata Mesajını Görmek (`Exception`)**

Hatanın teknik detayını öğrenmek istersek `catch` parantezinde bir değişken tanımlarız.

```csharp
try
{
    int a = 10;
    int b = 0;
    int sonuc = a / b;
}
catch (Exception ex) // 'ex' hatayı tutan nesnedir.
{
    Console.WriteLine("Bir hata oluştu.");
    // Sistemin orijinal hata mesajını yazdırır:
    Console.WriteLine($"Teknik Hata: {ex.Message}");
}
```
