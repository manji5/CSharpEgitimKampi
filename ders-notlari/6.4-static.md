# **Ders 27 Notları: `static` Üyeler ve `static` Sınıflar**

Selam Dostlar! Bu derste, `static` anahtar kelimesinin ne anlama geldiğini öğrendik.

### **1. `static` Olmayan (Instance) vs `static` Olan Üye**

- **1. Nesne Üyesi (Instance Member - `static` olmayan):**
  - `new` ile üretilen _her nesne için ayrıdır_.
  - Çağırmak için **nesne** gerekir: `ogr1.Ad`
  - `ogr1.Ad` ("Ali") ve `ogr2.Ad` ("Ayşe") birbirinden bağımsızdır.
- **2. `static` Üye (Durağan / Sınıf Üyesi):**
  - `new` ile üretilmez. **Sınıfın kendisine** aittir.
  - Tüm nesneler tarafından _paylaşılır_ ve _tektir_.
  - Çağırmak için **Sınıfın Adı** gerekir: `Math.PI`

| Özellik               | Nesne Üyesi (Instance)           | `static` Üye                            |
| --------------------- | -------------------------------- | --------------------------------------- |
| **Nasıl Tanımlanır?** | `public string Ad { get; set; }` | `public static int Sayac { get; set; }` |
| **Nasıl Çağrılır?**   | `Ogrenci ogr1 = new Ogrenci();`  |
| `ogr1.Ad = "Ali";`    | `Ogrenci.Sayac = 10;`            |
| **Hafıza**            | Her `new` ile yeni bir kopya     | Tüm program boyunca 1 tane              |

E-Tablolar'a aktar

**Altın Kural:** `static` bir metot (`static void Test()`), nesneye ait (`this.Ad` gibi) bir özelliği **kullanamaz!** Çünkü `static` metot, hangi nesneden bahsettiğinizi bilemez.

### **2. Pratik Örnek: Öğrenci Sayacı**

`static` bir değişken, tüm nesnelerin paylaştığı "ortak" bir veri tutmak için mükemmeldir.

```csharp
class Ogrenci
{
    // static: Tüm nesneler için ortak, tek bir sayaç.
    public static int ToplamOgrenciSayisi = 0;

    // Nesne üretilirken çalışan Constructor:
    public Ogrenci()
    {
        // Her yeni nesne üretildiğinde, ortak sayacı 1 artır.
        ToplamOgrenciSayisi++;
    }
}

// Main içinde kullanım:
Console.WriteLine(Ogrenci.ToplamOgrenciSayisi); // Çıktı: 0
Ogrenci ogr1 = new Ogrenci();
Console.WriteLine(Ogrenci.ToplamOgrenciSayisi); // Çıktı: 1
Ogrenci ogr2 = new Ogrenci();
Console.WriteLine(Ogrenci.ToplamOgrenciSayisi); // Çıktı: 2
```

### **3. `static` Sınıflar**

Eğer bir sınıfın içindeki **tüm metotlar ve özellikler `static`** ise, o sınıftan `new` ile nesne üretmek anlamsızdır. Bu durumda sınıfın kendisi de `static` olarak işaretlenir.

- **En İyi Örnekler:** `Math` ve `Console`
- `Math.Sqrt(9)` (Karekök), `Math.PI` (Pi sayısı)
- `Console.WriteLine("Merhaba")`, `Console.ReadLine()`

Bu sınıflar sadece "iş yapan" yardımcı metotların toplandığı kütüphaneler gibidir. `new Math()` veya `new Console()` yapmamıza gerek yoktur (ve yapamayız!).
