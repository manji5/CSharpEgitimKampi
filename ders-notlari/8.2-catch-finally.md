# **Ders 34 Notları: Birden Fazla catch ve finally**

Selam Dostlar! Bu derste, hataları türlerine göre ayırmayı ve temizlik yapmayı öğrendik.

### **1. Birden Fazla `catch` Bloğu**

Her hatanın bir türü vardır. Hataları özelleştirmek için birden fazla `catch` kullanabiliriz.

**Önemli Kural:** Sıralama **Özelden -> Genele** doğru olmalıdır. En spesifik hatalar üste, genel `Exception` en alta yazılır.

```csharp
try
{
    // Riskli kodlar
    int sayi = Convert.ToInt32(Console.ReadLine());
    int sonuc = 10 / sayi;
}
catch (DivideByZeroException) // Sadece 0'a bölme hatası
{
    Console.WriteLine("Sıfıra bölemezsiniz!");
}
catch (FormatException) // Sadece harf girme hatası
{
    Console.WriteLine("Lütfen sayı giriniz!");
}
catch (Exception ex) // Diğer tüm hatalar (Joker)
{
    Console.WriteLine($"Beklenmedik bir hata: {ex.Message}");
}
```

### **2. `finally` Bloğu**

Hata olsun ya da olmasın, `try-catch` yapısından çıkılmadan önce **mutlaka çalıştırılması gereken** kodlar buraya yazılır.

- **Kullanım Amacı:** Açık kalan dosyaları kapatmak, veritabanı bağlantılarını sonlandırmak, belleği temizlemek.

```csharp
try
{
    Console.WriteLine("Bağlantı Açıldı.");
    // Hata oluşabilecek işlemler...
    int x = 0;
    int y = 5 / x; // Hata!
}
catch
{
    Console.WriteLine("Hata oluştu.");
}
finally
{
    // Hata olsa da olmasa da bağlantıyı kapatmak zorundayız.
    Console.WriteLine("Bağlantı Kapatıldı (Temizlik Yapıldı).");
}
```
